## рҹҢҹ What is GenForge?

**GenForge** is a cutting-edge web application that fuses **military-grade encryption** with **AI-powered DNA optimization**. Whether you're a researcher safeguarding sensitive genomic data or exploring the intersection of bioinformatics and cybersecurity, GenForge delivers a seamless, secure, and intelligent pipeline for DNA sequence processing.

### рҹ”җ Encryption Layer

```python
# Payload structure: [ IV (16 bytes) | Ciphertext ]
# Random IV ensures each transmission is unique вҖ” even for identical sequences!

Plaintext DNA  в”Җв”Җв–ә  AES-CBC(key, IV)  в”Җв”Җв–ә  Base64  в”Җв”Җв–ә  Transmission
```

- **Algorithm**: AES (Advanced Encryption Standard)
- **Mode**: CBC (Cipher Block Chaining)
- **IV**: Cryptographically random 16-byte vector per transmission
- **Encoding**: Base64 for safe transport


Edit `config.py`:


## рҹ§Ә Usage Guide

### Input Rules

```
вң… Valid   : ATCGATCGATCGATCGATCG   (20 chars, A/T/C/G only)
вқҢ Invalid : ATCGATCGATCG           (too short)
вқҢ Invalid : ATCGATCGATCGATCGATCN   (invalid character 'N')
вқҢ Invalid : atcgatcgatcgatcgatcg   (lowercase not accepted)
```

### Example Workflow

```
INPUT  в”Җв”Җв–ә  ATCGATCGATCGATCGATCG
               в”Ӯ
               в–ј AES Encrypt
CIPHER в”Җв”Җв–ә  [IV][EncryptedData...] вҶ’ (Base64)
               в”Ӯ
               в–ј DNA-BERT Optimize
OUTPUT в”Җв”Җв–ә  ATCGTTCGATCGATCGAACG
               в”Ӯ
Changes:   Position 5: AвҶ’T | Position 18: TвҶ’C
Similarity:  Before: 0.847 | After: 0.923  рҹ“Ҳ
               в”Ӯ
               в–ј AES Encrypt
CIPHER в”Җв”Җв–ә  [IV][EncryptedResult...] вҶ’ (Sent back)
```

### Output Breakdown

| Field | Description |
|-------|-------------|
| рҹ”“ **Decrypted Sequence** | Your original DNA, confirmed received |
| вңҸпёҸ **Optimized Sequence** | AI-improved variant |
| рҹ”„ **Change Log** | Position-by-position mutation list |
| рҹ“Ҳ **Similarity Score (Before)** | Cosine similarity with reference |
| рҹ“Ҳ **Similarity Score (After)** | Improved score post-optimization |
| рҹ”җ **Encrypted Result** | Re-encrypted optimized sequence |

## вҡ пёҸ Known Limitations

> These are known trade-offs in the current academic implementation:

- рҹ”“ **No Message Authentication** вҖ” AES-CBC does not verify data integrity (no MAC/HMAC)
- рҹ“Ҹ **Fixed Input Length** вҖ” Sequences must be exactly 20 bases
- рҹ“ҡ **Static References** вҖ” Optimization targets are predefined reference sequences
- рҹ‘Ө **No Authentication** вҖ” No user login or session management
